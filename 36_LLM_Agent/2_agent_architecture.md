# 2. Agent架构与组件设计

## 整体架构

```
用户输入 → LLM(大脑) → 推理/规划 → 工具选择 → 执行 → 结果反馈
                ↕                                    ↕
             记忆系统 ←─────────────────────────────────┘
```

Agent本质上是一个循环系统：接收输入 → 思考 → 行动 → 观察结果 → 决定是否继续。

## 四大核心组件

### 1. LLM大脑
Agent的推理引擎和决策中心，负责：
- 理解用户意图
- 决定使用哪个工具
- 分析工具返回结果
- 生成最终回答

### 2. 工具系统（Tools）
Agent与外部世界交互的桥梁：
- 工具注册：声明可用工具及其描述
- 工具选择：LLM根据任务选择合适工具
- 工具执行：调用工具并获取结果

### 3. 记忆系统（Memory）
- **短期记忆**：当前对话的上下文（ConversationBufferMemory）
- **长期记忆**：持久化存储，通常用向量数据库（Chroma、FAISS等）
- **摘要记忆**：对长对话进行压缩摘要（ConversationSummaryMemory）

### 4. 规划器（Planner）
- 将复杂任务分解为子任务
- 制定执行计划
- 根据中间结果动态调整

## Agent执行循环

```
1. 接收用户输入
2. 分析任务意图
3. 决定行动策略（调用工具 or 直接回答）
4. 执行行动
5. 获取执行结果
6. 评估：任务完成了吗？
   - 是 → 返回最终结果
   - 否 → 回到步骤2继续
```

## Agent类型对比

| 类型 | 工作方式 | 适用场景 |
|------|---------|---------|
| 行动Agent（Action） | 每步直接决定下一个行动 | 简单任务 |
| 计划执行Agent（Plan-and-Execute） | 先制定完整计划，再逐步执行 | 复杂多步任务 |
| 反思Agent（Reflect） | 执行后自我反思并改进 | 需要高质量输出 |

## Prompt模板设计

Agent的Prompt需要告诉LLM：
1. 你是谁（角色设定）
2. 你有哪些工具可用（工具描述）
3. 你的工作流程（ReAct格式）
4. 输出格式要求

```
你是一个专业的AI助手。

可用工具:
- 搜索: 搜索实时信息，输入为关键词
- 计算器: 数学计算，输入为表达式

工作流程:
Thought: 分析问题，决定是否需要工具
Action: 工具名称
Action Input: 工具输入
Observation: 工具返回结果
... (重复直到得出答案)
Thought: 我已经有足够信息回答了
Final Answer: 最终回答
```

## 记忆系统详解

### 对话记忆
保存完整对话历史，适合短对话：
```python
ConversationBufferMemory()
```

### 摘要记忆
用LLM对历史对话做摘要，适合长对话：
```python
ConversationSummaryMemory(llm=llm)
```

### 向量记忆
将信息存入向量数据库，按相似度检索：
```python
VectorStoreRetrieverMemory(vectorstore=chroma_store)
```

## 错误处理策略

1. **工具执行失败** → try-except捕获，返回错误信息让LLM重新决策
2. **LLM输出格式错误** → 输出解析器自动修复或重试
3. **死循环** → 设置最大迭代次数（max_iterations）
4. **超时** → 设置最大执行时间（max_execution_time）

## 架构选择指南

| 任务复杂度 | 推荐架构 |
|-----------|---------|
| 简单问答+单工具 | 单Agent |
| 中等任务+多工具 | Agent + Tools |
| 复杂多步骤 | Plan-and-Execute Agent |
| 多角色协作 | Multi-Agent系统 |
