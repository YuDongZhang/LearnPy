# 1. Agent概述与核心概念

## 什么是LLM Agent

LLM Agent（智能代理）是基于大语言模型的自主系统，能够：
- 理解用户意图，自主规划任务步骤
- 调用外部工具（搜索、API、代码执行等）完成操作
- 具备记忆能力，支持多轮交互
- 根据执行结果动态调整策略

简单说：**传统LLM是被动问答，Agent是主动规划+行动。**

## Agent vs 传统LLM

| 方面 | 传统LLM | Agent |
|------|---------|-------|
| 交互方式 | 被动响应 | 主动规划+执行 |
| 工具使用 | 无 | 可调用外部工具 |
| 复杂任务 | 受限于单次推理 | 多步骤自主完成 |
| 记忆 | 仅上下文窗口 | 短期+长期记忆 |
| 适用场景 | 问答、文本生成 | 自动化、复杂任务 |

## Agent核心能力

### 推理能力
理解复杂问题，分解任务步骤，进行逻辑推理。

### 规划能力
制定执行计划，动态调整策略，将大目标拆解为小步骤。

### 工具使用
调用搜索引擎、API、代码执行器等外部工具，扩展LLM能力边界。

### 记忆系统
- 短期记忆：当前对话历史
- 长期记忆：外部向量数据库存储

## ReAct模式

ReAct（Reason + Act）是Agent的核心工作模式：

```
用户: 今天北京天气适合跑步吗？

Thought: 我需要先查询北京的天气信息
Action:  调用天气API，参数="北京"
Observation: 天气晴，25度，PM2.5=30

Thought: 天气数据显示条件不错，适合户外运动
Answer:  今天北京天气晴朗，温度25度，空气质量良好，很适合跑步。
```

每一轮循环：**思考(Thought) → 行动(Action) → 观察(Observation)**，直到得出最终答案。

## Agent类型

1. **单一Agent** — 处理单一类型任务，简单高效
2. **多Agent系统** — 多个Agent协作，角色分工，处理复杂任务
3. **Agent工作流** — 预定义流程+条件分支，自动化执行

## 应用场景

- 智能客服：自动问答+业务办理
- 个人助理：日程管理+信息检索
- 编程助手：代码生成+调试+执行
- 数据分析：数据处理+可视化
- 自动化工作流：定时任务+流程编排
