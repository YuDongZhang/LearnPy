# 1. Prompt基础与原理

## 什么是Prompt工程

Prompt工程是通过设计输入提示词来引导大语言模型产生期望输出的技术。不修改模型参数，仅通过"怎么问"来控制"怎么答"。

## 为什么Prompt很重要

同一个模型，不同的Prompt效果天差地别：

```
差的Prompt: "写代码"
好的Prompt: "用Python写一个快速排序函数，要求：1.包含类型注解 2.添加docstring 3.处理空列表边界情况"
```

## Prompt的组成要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 角色(Role) | 告诉模型扮演什么角色 | "你是一个资深Python开发者" |
| 任务(Task) | 明确要做什么 | "审查以下代码的安全问题" |
| 上下文(Context) | 提供背景信息 | "这是一个电商系统的支付模块" |
| 格式(Format) | 指定输出格式 | "用JSON格式输出" |
| 约束(Constraint) | 限制条件 | "回答不超过200字" |
| 示例(Example) | 给出参考样例 | Few-shot示例 |

## 好Prompt的原则

1. **具体明确** — 不要模糊，说清楚你要什么
2. **提供上下文** — 背景信息越充分，回答越准确
3. **指定格式** — 告诉模型用什么格式输出
4. **分步引导** — 复杂任务拆成步骤
5. **给出示例** — 用例子说明期望的输出

## System Prompt vs User Prompt

| 类型 | 作用 | 特点 |
|------|------|------|
| System Prompt | 设定角色和全局规则 | 对整个对话生效 |
| User Prompt | 具体的用户请求 | 每轮对话不同 |

```python
messages = [
    {"role": "system", "content": "你是一个Python专家，用简洁的中文回答"},
    {"role": "user", "content": "解释装饰器"}
]
```

## Temperature参数

控制输出的随机性：
- `temperature=0`：确定性输出，适合代码、事实问答
- `temperature=0.7`：平衡创造性和准确性
- `temperature=1.0+`：高创造性，适合创意写作
