# 2. 核心提示词技巧

## Zero-shot（零样本）

不给任何示例，直接提问：
```
将以下文本翻译成英文：今天天气很好
```
适合简单任务，模型已有足够知识。

## Few-shot（少样本）

给几个示例，让模型学习模式：
```
将中文翻译成英文：
中文：你好 → 英文：Hello
中文：谢谢 → 英文：Thank you
中文：今天天气很好 → 英文：
```
适合需要特定格式或风格的任务。

## Chain-of-Thought（思维链，CoT）

让模型一步步推理，而不是直接给答案：
```
问：一个商店有15个苹果，卖出8个，又进货12个，现在有多少？
请一步步思考：
```

### Zero-shot CoT
只需加一句"让我们一步步思考"：
```
问题：...
让我们一步步思考。
```

### Few-shot CoT
给出带推理过程的示例。

## Self-Consistency（自一致性）

多次采样（高temperature），取多数一致的答案。适合数学推理等需要准确性的场景。

## ReAct（推理+行动）

结合推理和工具调用：
```
Thought: 我需要查询天气信息
Action: search("北京天气")
Observation: 北京今天晴，25度
Thought: 天气不错，适合户外
Answer: 今天北京天气晴朗，25度，适合户外活动。
```
这是Agent的核心模式。

## Tree-of-Thought（思维树，ToT）

探索多条推理路径，评估每条路径的可行性，选择最优路径。适合复杂决策问题。

## 角色扮演

给模型设定专业角色：
```
你是一个有20年经验的数据库架构师。
请评审以下SQL查询的性能问题...
```
角色越具体，回答越专业。

## 技巧对比

| 技巧 | 复杂度 | 效果 | 适用场景 |
|------|--------|------|---------|
| Zero-shot | 低 | 一般 | 简单任务 |
| Few-shot | 中 | 好 | 格式/风格控制 |
| CoT | 中 | 很好 | 推理、数学 |
| ReAct | 高 | 很好 | 需要工具的任务 |
| ToT | 高 | 最好 | 复杂决策 |
