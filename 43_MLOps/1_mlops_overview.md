# 1. MLOps概述

## 什么是MLOps

MLOps = Machine Learning + DevOps。将软件工程的最佳实践应用到机器学习项目中，实现模型的自动化开发、部署和运维。

## 为什么需要MLOps

没有MLOps的痛点：
- 模型训练不可复现（"我的电脑上能跑"）
- 不知道哪个版本的模型在线上
- 手动部署，容易出错
- 模型效果退化没人知道

## MLOps生命周期

```
数据准备 → 模型训练 → 模型评估 → 模型注册 → 部署上线 → 监控 → 反馈
    ↑                                                          |
    └──────────────────────────────────────────────────────────┘
```

## 核心实践

| 实践 | 说明 | 工具 |
|------|------|------|
| 版本管理 | 代码+数据+模型都要版本化 | Git、DVC、MLflow |
| 实验追踪 | 记录每次训练的参数和结果 | MLflow、W&B |
| 自动化流水线 | 数据→训练→评估→部署自动化 | Airflow、GitHub Actions |
| 模型注册 | 统一管理模型版本和状态 | MLflow Model Registry |
| A/B测试 | 新旧模型对比验证 | 自建或平台支持 |
| 持续监控 | 线上效果持续追踪 | Prometheus、自定义 |

## MLOps成熟度

| 级别 | 说明 | 特征 |
|------|------|------|
| Level 0 | 手动 | 手动训练、手动部署 |
| Level 1 | 流水线 | 自动化训练流水线 |
| Level 2 | CI/CD | 自动化训练+自动化部署 |

## LLM时代的MLOps

LLM项目的MLOps有些不同：
- 模型通常不自己训练（用API或开源模型）
- 重点在Prompt管理、RAG数据管理
- 评估更依赖LLM-as-Judge
- 版本管理包括Prompt版本、数据版本
