# 3. 自动化流水线

## 什么是ML流水线

将数据准备→训练→评估→部署串成自动化流程：

```
数据更新 → 触发流水线 → 数据预处理 → 训练 → 评估 → 通过? → 部署
                                                    ↓ 不通过
                                                  告警通知
```

## 流水线步骤

| 步骤 | 输入 | 输出 |
|------|------|------|
| 数据预处理 | 原始数据 | 清洗后的数据集 |
| 训练 | 数据集+配置 | 模型权重 |
| 评估 | 模型+测试集 | 评估指标 |
| 注册 | 模型+指标 | 注册到Model Registry |
| 部署 | 注册的模型 | 线上服务更新 |

## 触发方式

| 方式 | 说明 | 适用 |
|------|------|------|
| 手动触发 | 人工启动 | 开发阶段 |
| 定时触发 | 每天/每周 | 定期重训练 |
| 数据触发 | 新数据到达时 | 数据驱动 |
| 代码触发 | 代码合并时 | CI/CD |

## 工具选择

| 工具 | 特点 | 适用 |
|------|------|------|
| GitHub Actions | 与Git集成，免费额度 | 小团队 |
| Airflow | 复杂DAG调度 | 大规模 |
| Prefect | 现代Python原生 | 中等规模 |
| DVC Pipeline | 与DVC集成 | 数据驱动 |

## LLM项目的流水线

LLM项目的流水线和传统ML不同：
```
Prompt更新 → 评估（LLM-as-Judge）→ 通过? → 更新线上Prompt
数据更新 → 重建RAG索引 → 评估检索质量 → 更新向量库
```
