# 4. 模型评估与A/B测试

## 离线评估

上线前用测试集评估：
- 准确率、F1、BLEU等传统指标
- LLM-as-Judge：用GPT-4评判回答质量
- 人工评估：抽样人工打分

## 在线评估（A/B测试）

新旧模型同时服务，对比效果：
```
用户请求 → 路由（50%/50%）→ 模型A（旧）
                            → 模型B（新）
收集指标 → 统计对比 → 决定是否全量切换
```

### 关键指标
- 用户满意度（点赞/点踩）
- 回答准确性
- 响应时间
- Token消耗

### 统计显著性
用t检验或卡方检验判断差异是否显著，避免被随机波动误导。

## 回归测试

每次更新后跑固定测试集，确保没有退化：
```python
def test_basic_qa():
    assert "编程语言" in ask("Python是什么？")

def test_no_hallucination():
    answer = ask("2024年奥运会在哪里举办？")
    assert "巴黎" in answer
```

## 持续监控

上线后持续追踪：
- 用户反馈（好评率）
- 错误率
- 幻觉率
- 拒绝回答率
