# 2. 缓存策略

## 为什么AI应用需要缓存

LLM调用特点：慢（1-10秒）、贵（按Token计费）。缓存可以：
- 相同问题秒回
- 大幅降低API成本
- 减轻LLM服务压力

## 缓存层次

| 层次 | 说明 | 命中率 |
|------|------|--------|
| 精确缓存 | 完全相同的问题 | 低但精准 |
| 语义缓存 | 语义相似的问题 | 高 |
| 结果缓存 | 工具调用结果 | 中 |

## 精确缓存

用问题的hash作为key：
```python
cache_key = hashlib.md5(question.encode()).hexdigest()
```
简单但命中率低（"什么是GIL"和"GIL是什么"就不命中）。

## 语义缓存（推荐）

用Embedding计算语义相似度：
1. 新问题 → Embedding
2. 在缓存中找最相似的问题
3. 相似度超过阈值 → 返回缓存答案
4. 否则 → 调用LLM，结果存入缓存

## 缓存失效策略

| 策略 | 说明 |
|------|------|
| TTL | 设置过期时间（如1小时） |
| LRU | 淘汰最久未使用的 |
| 容量限制 | 超过上限淘汰旧数据 |
| 手动失效 | 数据更新时主动清除 |

## Redis缓存

生产环境推荐Redis：
- 高性能（内存存储）
- 支持TTL
- 支持分布式
- 丰富的数据结构
