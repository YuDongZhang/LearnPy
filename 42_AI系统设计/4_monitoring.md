# 4. 监控与日志

## 监控三大支柱

| 支柱 | 说明 | 工具 |
|------|------|------|
| 指标(Metrics) | 数值型数据，趋势分析 | Prometheus + Grafana |
| 日志(Logs) | 事件记录，问题排查 | 结构化日志 + ELK |
| 追踪(Traces) | 请求链路，性能分析 | LangSmith / Jaeger |

## AI应用关键指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| 请求QPS | 每秒请求数 | 超过容量80% |
| 响应时间P99 | 99%请求的耗时 | >10秒 |
| 错误率 | 失败请求占比 | >5% |
| Token消耗 | 每日Token用量 | 超过预算 |
| GPU利用率 | GPU使用率 | >90%持续5分钟 |
| 缓存命中率 | 缓存命中占比 | <30% |

## 结构化日志

用JSON格式记录日志，方便检索和分析：
```json
{
  "timestamp": "2024-01-01T12:00:00",
  "level": "INFO",
  "user": "demo",
  "action": "chat",
  "tokens": 150,
  "latency_ms": 2300,
  "model": "qwen2.5:7b"
}
```

## Prometheus指标

用prometheus_client库暴露指标：
- Counter：计数器（请求总数、错误总数）
- Histogram：直方图（响应时间分布）
- Gauge：仪表盘（当前并发数）

## 告警规则

- 错误率 > 5% 持续5分钟 → 紧急
- P99延迟 > 10秒 持续10分钟 → 警告
- 日Token消耗 > 预算80% → 通知
- 服务不可用 → 紧急
