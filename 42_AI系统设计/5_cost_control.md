# 5. 成本控制

## AI应用的成本构成

| 成本项 | 占比 | 说明 |
|--------|------|------|
| LLM推理 | 60-80% | Token费用或GPU租赁 |
| 向量数据库 | 5-15% | Embedding + 存储 |
| 基础设施 | 10-20% | 服务器、带宽 |
| 人力 | 变动 | 开发维护 |

## 降低LLM成本

### 1. 缓存
语义缓存可减少30-60%的LLM调用。

### 2. 模型选择
- 简单任务用小模型（Phi-3、Qwen-1.5B）
- 复杂任务才用大模型（GPT-4o、Qwen-72B）
- 路由：先判断任务复杂度，再选模型

### 3. Prompt优化
- 精简System Prompt
- 控制max_tokens
- 减少不必要的上下文

### 4. 本地部署
- 自建GPU服务器（长期更便宜）
- 用量化模型降低GPU需求

## 成本监控

记录每次调用的Token数和费用：
```python
cost = (input_tokens * input_price + output_tokens * output_price) / 1000
```

## 用量控制

- 按用户设置每日/每月Token上限
- 超限后降级（用小模型）或拒绝
- 提供用量仪表盘

## 模型路由

根据任务复杂度自动选择模型：
```
简单问答 → Qwen-1.5B（便宜）
复杂推理 → GPT-4o（贵但好）
代码生成 → DeepSeek-Coder（专业）
```
