# 5. RLHF与DPO对齐

## 为什么需要对齐

预训练+SFT后的模型可能：
- 输出有害内容
- 不遵循用户意图
- 编造信息（幻觉）

对齐让模型输出更安全、更有用、更诚实。

## RLHF（基于人类反馈的强化学习）

### 三步流程

```
Step 1: SFT — 指令微调得到初始模型
Step 2: 训练奖励模型(RM)
  - 人类对多个回答排序
  - 训练模型预测人类偏好
Step 3: PPO强化学习
  - 用RM作为奖励信号
  - PPO算法优化策略模型
```

### 奖励模型
输入：(prompt, response) → 输出：分数
```
prompt: "写一首诗"
response_A: "春眠不觉晓..." → 分数: 0.9
response_B: "我不会写诗" → 分数: 0.2
```

### RLHF的问题
- 训练不稳定
- 需要4个模型（策略、参考、奖励、价值）
- 超参数敏感
- 成本高

## DPO（直接偏好优化）

### 核心思想
跳过奖励模型，直接用偏好数据优化：

```
数据格式:
{
  "prompt": "写一首诗",
  "chosen": "春眠不觉晓...",    # 好的回答
  "rejected": "我不会写诗"       # 差的回答
}
```

### DPO vs RLHF

| 特性 | RLHF | DPO |
|------|------|-----|
| 复杂度 | 高（4个模型） | 低（1个模型） |
| 稳定性 | 差 | 好 |
| 效果 | 很好 | 接近RLHF |
| 成本 | 高 | 低 |
| 当前趋势 | 逐渐被替代 | 主流 |

## 其他对齐方法

| 方法 | 说明 |
|------|------|
| ORPO | 不需要参考模型的DPO变体 |
| SimPO | 简化版DPO |
| KTO | 只需要好/坏标签，不需要配对 |
| Constitutional AI | Anthropic的自我改进方法 |
