# 1. 多Agent系统设计

## 什么是多Agent系统

多Agent系统（Multi-Agent System）是多个Agent协同工作来完成复杂任务的架构。每个Agent有自己的角色、工具和职责，通过协作完成单个Agent难以胜任的工作。

## 为什么需要多Agent

| 单Agent | 多Agent |
|---------|---------|
| 一个Agent处理所有事 | 专业分工，各司其职 |
| 复杂任务容易出错 | 模块化，错误隔离 |
| 难以扩展 | 灵活扩展新角色 |
| Prompt越来越长 | 每个Agent的Prompt简洁聚焦 |

## 常见角色设计

| 角色 | 职责 | 典型场景 |
|------|------|---------|
| 规划者(Planner) | 分解任务、制定计划 | 复杂任务入口 |
| 执行者(Executor) | 具体执行子任务 | 搜索、编码、写作 |
| 审核者(Reviewer) | 检查结果质量 | 质量把关 |
| 协调者(Coordinator) | 协调多Agent流程 | 任务分发和汇总 |

## 协作模式

### 串行协作
```
Agent1 → Agent2 → Agent3 → 结果
```
任务按顺序流转，下游Agent使用上游输出。适合流水线式任务。

### 并行协作
```
        ┌→ Agent1 ─┐
任务 ───┼→ Agent2 ──┼→ 汇总 → 结果
        └→ Agent3 ─┘
```
多个Agent同时处理子任务，最后汇总。适合可拆分的独立子任务。

### 循环协作（辩论/迭代）
```
Agent1 ←→ Agent2（反复交互直到达成共识）
```
Agent之间反复交互、修改、审核，直到结果满意。适合需要打磨质量的场景。

## Agent通信

Agent之间需要传递信息，常见方式：
1. **函数调用** — 直接把上一个Agent的输出作为下一个的输入（最简单）
2. **消息队列** — 通过消息对象传递，包含发送者、接收者、内容、类型
3. **共享状态** — 所有Agent读写同一个状态字典（LangGraph的方式）

## 冲突处理

多Agent可能产生分歧，常见解决方式：
- **投票机制** — 多个Agent投票，少数服从多数
- **仲裁者** — 指定一个高权限Agent做最终决策
- **迭代修正** — 反复修改直到所有Agent都满意

## 设计原则

1. 角色职责清晰，避免重叠
2. 每个Agent的Prompt简洁聚焦
3. 定义好Agent之间的数据格式
4. 设置最大迭代次数，防止死循环
5. 做好错误隔离，一个Agent失败不影响整体
