# 5. Agent评估与测试

## 为什么要评估Agent

Agent的行为具有不确定性，同样的输入可能产生不同的输出。评估帮助我们：
- 量化Agent的性能表现
- 发现问题并持续改进
- 对比不同方案的优劣
- 确保生产环境的可靠性

## 评估维度

| 维度 | 说明 | 常用指标 |
|------|------|---------|
| 任务完成度 | 是否完成了目标 | 成功率、完成率 |
| 准确性 | 结果是否正确 | 准确率、语义相似度 |
| 效率 | 资源消耗情况 | LLM调用次数、耗时、Token数 |
| 稳定性 | 多次运行结果是否一致 | 方差、一致性比率 |
| 成本 | 花了多少钱 | Token费用、API调用费 |
| 安全性 | 是否产生有害输出 | 安全评分 |

## 评估方法

### 自动评估
- 用预定义的测试用例跑Agent，对比期望输出
- 用LLM作为评判者（LLM-as-Judge）打分
- 计算语义相似度（Embedding余弦相似度）

### 人工评估
- 人工对Agent输出打分（准确性、有用性、清晰度等）
- 适合主观性强的任务
- 成本高但最可靠

### A/B测试
- 两个Agent方案同时运行
- 对比性能指标
- 用统计检验判断差异是否显著

## 测试策略

### 单元测试
测试单个工具或节点是否正常工作。

### 集成测试
测试完整的Agent流程，验证端到端行为。

### 回归测试
每次修改后重跑测试集，确保没有引入新问题。

### 压力测试
大量并发请求，测试Agent在高负载下的表现。

## 常见问题与优化

| 问题 | 可能原因 | 优化方向 |
|------|---------|---------|
| 任务完成率低 | 工具描述不清晰 | 优化Prompt和工具描述 |
| 调用次数过多 | Agent在循环 | 设置max_iterations |
| 结果不稳定 | temperature太高 | 降低temperature |
| 成本过高 | 不必要的LLM调用 | 缓存、减少调用 |
| 选错工具 | 工具描述有歧义 | 改进工具描述 |

## 持续监控

生产环境需要持续监控：
- 成功率趋势
- 平均响应时间
- Token消耗和成本
- 异常和错误率
- 性能退化告警
