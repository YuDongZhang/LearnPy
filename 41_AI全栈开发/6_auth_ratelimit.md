# 6. 认证与限流

## 为什么需要认证和限流

- 防止未授权访问
- 防止滥用（LLM调用有成本）
- 按用户计费
- 保护系统稳定性

## 认证方案

### API Key
最简单，适合内部服务和开发者API：
```
Authorization: Bearer sk-xxxx
```

### JWT Token
适合有用户系统的应用：
```
1. 用户登录 → 返回JWT Token
2. 后续请求带Token → 验证身份
```

### OAuth2
适合第三方登录（Google、GitHub等）。

## 限流策略

| 策略 | 说明 | 适用 |
|------|------|------|
| 固定窗口 | 每分钟N次 | 简单场景 |
| 滑动窗口 | 更平滑的限制 | 生产环境 |
| 令牌桶 | 允许突发流量 | API网关 |
| 按用户限流 | 每个用户独立限制 | 多用户系统 |

## 实现方式

- FastAPI中间件 — 全局限流
- 依赖注入 — 按路由限流
- Redis — 分布式限流
- slowapi库 — 开箱即用

## 成本控制

LLM调用有成本，需要：
- 记录每次调用的Token数
- 按用户统计用量
- 设置用量上限
- 超限后降级或拒绝
