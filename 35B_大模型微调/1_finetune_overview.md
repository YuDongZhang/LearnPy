# 1. 微调概述与方法对比

## 什么是微调

微调（Fine-tuning）是在预训练好的大模型基础上，用特定领域或任务的数据继续训练，让模型适应你的具体需求。

```
预训练模型（通用能力）+ 你的数据 → 微调 → 领域专用模型
```

## 为什么需要微调

| 场景 | 说明 |
|------|------|
| 领域适配 | 让模型掌握医疗、法律、金融等专业知识 |
| 风格定制 | 让模型按特定风格/格式输出 |
| 指令遵循 | 提升模型对指令的理解和执行能力 |
| 私有数据 | 让模型学习企业内部知识 |
| 成本优化 | 小模型微调后可替代大模型API |

## 微调 vs Prompt工程 vs RAG

| 方案 | 原理 | 优势 | 劣势 |
|------|------|------|------|
| Prompt工程 | 通过提示词引导模型 | 零成本、即时生效 | 能力有限、受上下文窗口限制 |
| RAG | 检索外部知识+生成 | 知识可更新、无需训练 | 依赖检索质量、延迟较高 |
| 微调 | 修改模型参数 | 深度定制、推理快 | 需要数据和算力、知识不易更新 |

> 实际项目中三者经常组合使用。

## 微调方法分类

### 全量微调（Full Fine-tuning）
更新模型所有参数。效果最好，但需要巨大显存和算力。

### 参数高效微调（PEFT）
只训练少量参数，冻结大部分原始参数：

| 方法 | 原理 | 参数量 | 显存需求 |
|------|------|--------|---------|
| LoRA | 在权重矩阵旁加低秩适配矩阵 | ~0.1% | 低 |
| QLoRA | LoRA + 4bit量化 | ~0.1% | 很低 |
| P-Tuning v2 | 在每层加可训练前缀 | ~1% | 低 |
| Prefix Tuning | 在注意力层加前缀向量 | ~1% | 低 |
| Adapter | 在层间插入小型网络 | ~1-5% | 中 |

### 当前主流：LoRA / QLoRA
LoRA是目前最主流的微调方案，QLoRA进一步降低了显存需求，让消费级显卡也能微调7B甚至13B模型。

## 微调流程

```
1. 选择基座模型（Base Model）
2. 准备训练数据（指令格式）
3. 选择微调方法（LoRA/QLoRA/全量）
4. 配置训练参数
5. 开始训练
6. 评估效果
7. 合并权重 & 部署
```
