# 3. 全量微调

## 什么是全量微调

全量微调（Full Fine-tuning）更新模型的所有参数，是最传统也是效果上限最高的微调方式。

## 优缺点

| 优势 | 劣势 |
|------|------|
| 效果最好，模型充分适配 | 显存需求巨大 |
| 适合大规模数据 | 训练时间长 |
| 无额外推理开销 | 容易过拟合（数据少时） |
| | 每个任务需要保存完整模型副本 |

## 显存估算

全量微调一个7B模型大约需要：
- 模型参数：7B × 2字节(fp16) = 14GB
- 梯度：14GB
- 优化器状态(Adam)：28GB
- 激活值：~10-20GB
- **总计：约60-80GB**

> 这就是为什么全量微调通常需要A100(80GB)或多卡并行。

## 关键训练参数

| 参数 | 说明 | 常用值 |
|------|------|--------|
| learning_rate | 学习率 | 1e-5 ~ 5e-5 |
| num_train_epochs | 训练轮数 | 1-3 |
| per_device_train_batch_size | 每卡批次大小 | 1-4 |
| gradient_accumulation_steps | 梯度累积步数 | 4-16 |
| warmup_ratio | 预热比例 | 0.03-0.1 |
| weight_decay | 权重衰减 | 0.01-0.1 |
| max_grad_norm | 梯度裁剪 | 1.0 |
| fp16/bf16 | 混合精度训练 | True |

## 训练流程

```
1. 加载预训练模型和Tokenizer
2. 加载并预处理数据集
3. 配置TrainingArguments
4. 创建Trainer
5. 开始训练
6. 评估和保存模型
```

## 什么时候用全量微调

- 有充足的算力（多卡A100）
- 数据量大（>10000条）
- 需要最好的效果
- 模型较小（<3B参数）

## 什么时候不用

- 显存不够 → 用LoRA/QLoRA
- 数据量少（<1000条）→ 容易过拟合
- 需要快速实验 → LoRA更快
- 需要多个任务版本 → LoRA适配器更灵活
