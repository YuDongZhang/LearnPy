# 1. 部署概述与方案对比

## 为什么要学部署

模型训练/微调完成后，需要部署为可用的服务。部署决定了：
- 用户能不能用上你的模型
- 推理速度快不快
- 成本高不高

## 部署方案对比

| 方案 | 特点 | 适用场景 |
|------|------|---------|
| Ollama | 一键部署，最简单 | 本地开发、个人使用 |
| vLLM | 高性能，支持并发 | 生产环境、多用户 |
| TGI | HuggingFace出品 | 生产环境 |
| llama.cpp | CPU推理，跨平台 | 无GPU环境 |
| Transformers | 最灵活 | 开发测试 |

## 量化技术

将模型从fp16压缩到更低精度，减少显存和加速推理：

| 格式 | 精度 | 特点 |
|------|------|------|
| FP16 | 16bit | 原始精度，显存大 |
| GPTQ | 4bit | GPU推理，速度快 |
| AWQ | 4bit | GPU推理，质量略好于GPTQ |
| GGUF | 2-8bit | CPU/GPU混合，Ollama使用 |

## 部署架构

```
客户端 → API网关 → 推理服务(vLLM/Ollama) → 模型
                      ↕
                   模型文件(量化后)
```

## 选择建议

```
个人学习/开发 → Ollama（最简单）
生产环境/多用户 → vLLM（最快）
无GPU → llama.cpp / Ollama(CPU模式)
需要最大灵活性 → Transformers
```
