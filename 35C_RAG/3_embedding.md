# 3. Embedding与向量数据库

## 什么是Embedding

Embedding是将文本转换为高维向量的过程。语义相似的文本，向量距离也近。

```
"Python是编程语言" → [0.12, -0.34, 0.56, ...]  (1536维)
"Java是编程语言"   → [0.11, -0.33, 0.55, ...]  (很接近)
"今天天气很好"     → [0.78, 0.21, -0.45, ...]  (很远)
```

## 常用Embedding模型

| 模型 | 维度 | 特点 |
|------|------|------|
| OpenAI text-embedding-3-small | 1536 | 效果好，需API |
| BGE-large-zh | 1024 | 中文最佳开源 |
| M3E | 768 | 中文开源，轻量 |
| all-MiniLM-L6-v2 | 384 | 英文轻量 |

## 向量数据库

存储Embedding向量并支持相似度检索：

| 数据库 | 特点 | 适用场景 |
|--------|------|---------|
| Chroma | 轻量、内嵌、易上手 | 开发测试、小项目 |
| FAISS | Meta出品、速度快 | 大规模、纯本地 |
| Milvus | 分布式、生产级 | 大规模生产环境 |
| Pinecone | 云托管、免运维 | 不想自己运维 |
| Weaviate | 支持混合搜索 | 需要关键词+向量混合 |

## 相似度计算

| 方法 | 说明 |
|------|------|
| 余弦相似度 | 最常用，计算向量夹角 |
| 欧氏距离 | 计算向量间的直线距离 |
| 内积 | 点积，适合归一化向量 |

## 使用流程

```
1. 选择Embedding模型
2. 将文档块转为向量
3. 存入向量数据库
4. 查询时：问题 → 向量 → 检索Top-K相似文档
```
