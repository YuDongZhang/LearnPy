# 1. RAG概述与原理

## 什么是RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是将外部知识检索与LLM生成相结合的技术。

```
用户提问 → 检索相关文档 → 将文档+问题一起给LLM → 生成回答
```

## 为什么需要RAG

LLM的局限性：
- 知识有截止日期，不知道最新信息
- 不了解你的私有数据（企业文档、内部知识）
- 可能产生幻觉（编造不存在的信息）

RAG的解决方案：
- 从外部知识库检索真实信息
- 让LLM基于检索到的内容回答
- 大幅减少幻觉，答案有据可查

## RAG vs 微调

| 对比 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 实时更新，改文档即可 | 需要重新训练 |
| 成本 | 低，不需要GPU训练 | 高，需要算力 |
| 准确性 | 高，基于真实文档 | 取决于训练数据 |
| 可解释性 | 好，可追溯来源 | 差，黑盒 |
| 适用场景 | 知识问答、文档助手 | 风格定制、能力增强 |

## RAG完整流程

### 离线阶段（索引构建）
```
文档 → 加载 → 切分(Chunk) → Embedding向量化 → 存入向量数据库
```

### 在线阶段（问答）
```
用户提问 → Embedding → 向量检索 → 取出Top-K文档 → 拼接Prompt → LLM生成回答
```

## 核心组件

| 组件 | 作用 | 常用工具 |
|------|------|---------|
| 文档加载器 | 读取各种格式文档 | LangChain Loaders |
| 文本切分器 | 将长文档切成小块 | RecursiveCharacterTextSplitter |
| Embedding模型 | 将文本转为向量 | OpenAI、BGE、M3E |
| 向量数据库 | 存储和检索向量 | Chroma、FAISS、Milvus |
| LLM | 基于检索内容生成回答 | GPT-4、Qwen等 |

## 应用场景

- 企业知识库问答
- 文档助手（PDF、Word）
- 客服系统
- 代码库问答
- 法律/医疗/金融领域问答
