# 6. RAG优化技巧

## 优化方向

RAG的效果取决于三个环节：**切分质量 × 检索质量 × 生成质量**。

## 切分优化

| 技巧 | 说明 |
|------|------|
| 语义切分 | 按语义边界而非固定长度切分 |
| 保留元数据 | 记录来源文件、页码、标题等 |
| 小块检索+大块生成 | 用小块做检索（精准），返回时扩展到大块（上下文完整） |
| 按文档结构切分 | Markdown按标题、代码按函数 |

## 检索优化

| 技巧 | 说明 |
|------|------|
| 混合检索 | BM25关键词 + 向量语义，互补 |
| 查询改写 | 用LLM改写用户问题，提高检索命中 |
| HyDE | 先让LLM生成假设答案，用假设答案做检索 |
| 重排序(Rerank) | 检索后用Rerank模型重新排序 |
| 多路召回 | 多种检索方式并行，合并去重 |

## 生成优化

| 技巧 | 说明 |
|------|------|
| Prompt优化 | 明确要求基于上下文回答 |
| 来源标注 | 要求LLM标注信息来源 |
| 拒绝回答 | 上下文无关时说"不知道" |
| 压缩上下文 | 只保留与问题相关的部分 |

## 评估指标

| 指标 | 评估什么 |
|------|---------|
| 检索命中率 | 正确文档是否被检索到 |
| 回答准确性 | 回答是否正确 |
| 忠实度 | 回答是否基于检索内容（非幻觉） |
| 相关性 | 回答是否与问题相关 |

## 常用评估工具

- **RAGAS** — 自动化RAG评估框架
- **LangSmith** — LangChain的监控和评估平台
- **人工评估** — 最可靠但成本高
